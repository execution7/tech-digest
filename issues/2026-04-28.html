🔥 头条深读

OpenAI 与微软今天宣布修订合作协议。微软对 OpenAI 的知识产权许可变为非独占（有效期至 2032 年），OpenAI 可在任何云平台服务客户，Azure 仅保留「首选云合作伙伴」地位。OpenAI 向微软支付的收入分成维持 20%，但设定上限、2030 年后终止，且——这是关键——「与 OpenAI 的技术进展无关」。

Simon Willison 一针见血：「据我所知，『与 OpenAI 的技术进展无关』就是在宣告 AGI 条款已死。」那个曾让整个行业幻想「一旦实现 AGI，所有协议作废」的神秘条款，在商业现实面前寿终正寝。其背景是 OpenAI 与亚马逊高达 500 亿美元的云服务协议——微软此前的独占权成了交易的绊脚石。

同日，GitHub 宣布 Copilot 将于 6 月 1 日全面转向按量计费，用 AI Credits 替代 Premium Request。理由很直白：「Agent 式用量正在成为默认模式，带来的计算和推理开销远超以往。」一次闲聊和一次多小时的自主编码会话，不应该花一样的钱。Pro 用户每月 10 美元含 10 美元 Credits，Enterprise 用户 39 美元含 39 美元 Credits，代码补全和 Next Edit 不消耗额度。

判断：独占时代结束，补贴时代终结。当 AI 从「花别人的钱跑」变成「花自己的钱跑」，团队要做的第一件事不是选模型，而是建立 token 用量监控和成本预算体系。接下来几个月，预计会看到更多 AI 产品从固定订阅转向按量计费。

对团队的启发：立即评估团队 Copilot / Cursor / Claude Code 的实际 token 消耗量，在 6 月 1 日前建立用量基线和预算上限。多云部署也意味着 OpenAI 模型将出现在 AWS Bedrock 上——API 选型多了一个变量。

📰 科技动态

1、GitHub Copilot 6 月 1 日起全面转向按量计费 confirmed

GitHub 宣布所有 Copilot 计划将用 AI Credits（按 token 计费）替代 Premium Request。订阅价格不变，代码补全和 Next Edit 免费，但 Agent 模式、多模型切换等高级用量按实际消耗扣费。5 月初上线计费预览。理由：Agent 式用量正在成为默认模式，固定价格无法覆盖差异化算力需求。

2、中国叫停 Meta 20 亿美元收购 AI 创业公司 Manus confirmed

中国发改委要求 Meta 和 Manus 撤回收购交易，称依据相关法律法规对外商投资进行合规审查。Manus 是一家总部位于新加坡、有中国背景的 AI Agent 公司，曾获 Benchmark 领投 7500 万美元融资，声称 ARR 达 1 亿美元。该事件反映了「新加坡洗白」和中美 AI 投资限制的持续紧张。

3、DeepMind 强化学习之父 David Silver 创办 Ineffable Intelligence，融资 11 亿美元 confirmed

Silver 创立的 Ineffable Intelligence 以 51 亿美元估值融资 11 亿美元，目标是构建不依赖人类数据的「超级学习器」，纯靠强化学习训练。投资方包括 Sequoia、Lightspeed、Google、Nvidia 和英国主权 AI 基金。Silver 称这是「他毕生的事业」，并承诺个人收入全部捐给慈善机构。同期，AMI Labs 融 10.3 亿，Recursive Superintelligence 据报融资需求达 10 亿。

4、Mercor 4TB 语音数据泄露：4 万 AI 承包商的声纹被盗 confirmed

Lapsus$ 在泄露站点公开了 AI 数据标注平台 Mercor 的约 4TB 数据，涉及 4 万余名承包商的政府证件、自拍和长时间录音室语音样本。分析指出 Mercor 的入职流程将身份证件和高质量语音绑定在一起，为声纹克隆提供了完美素材——多数商业克隆工具只需约 15 秒干净音频，而 Mercor 样本平均 2-5 分钟。已有 5 起承包商诉讼。建议：把声纹当作泄露的密码对待——你无法轮换它，但可以改变它能解锁的东西。

5、pgBackRest 仓库已归档，不再维护 confirmed

PostgreSQL 备份恢复工具 pgBackRest 的 GitHub 仓库于 4 月 27 日被所有者归档为只读。这个拥有约 3800 Star、支持并行备份/恢复、多仓库、块级增量、S3/Azure/GCS 等特性的项目，是许多企业 PostgreSQL 基础设施的核心组件。Issues 和 PR 已冻结，依赖该工具的团队需确认迁移路径。

6、Python 包 element-data 供应链攻击：月下载量百万级 confirmed

ML 监控工具 Elementary 的 CLI 包 element-data 遭供应链攻击。攻击者利用 GitHub Action 工作流漏洞，通过恶意 PR 在维护者上下文中执行脚本，窃取签名密钥后发布恶意版本 0.23.3 至 PyPI 和 Docker。该版本会搜集云密钥、API Token、SSH 密钥等凭证。恶意版本约 12 小时后被移除，受影响用户应假定凭证已暴露并立即轮换。runZero 的 HD Moore 指出：「很难不在无意中创建可被攻击者 PR 利用的危险工作流。」

7、FDA 批准首个基因疗法治疗遗传性听力损失 confirmed

FDA 批准 Regeneron 的 Otarmeni（lunsotogene parvec-cwha），这是首个针对双等位 OTOF 基因变异导致的重度至极重度感音神经性耳聋的基因疗法。该生物制品-器械组合通过手术递送至耳蜗，在 24 名儿童患者的试验中，80% 的可评估患者听力改善。通过国家优先券通道，从申请到批准仅 61 天——创现代 BLA 审查速度纪录。

Anthropic 在旧金山办公室开展了一场为期一周的内部实验：69 名员工让 Claude Agent 在 Slack 上代表自己谈判二手物品交易，每人 100 美元预算，共完成 186 笔交易、总额超 4000 美元。关键发现：秘密 A/B 测试显示，使用 Opus 4.5 的用户获得了客观上更好的交易结果，但使用 Haiku 4.5 的用户并未察觉自己处于劣势。这暗示了 AI Agent 商务谈判中的信息不对称问题。

2、81,000 人告诉 Anthropic 关于 AI 经济学的真相 confirmed

Anthropic 经济研究团队分析了 81,000 名 Claude 用户的开放式访谈。约 1/5 的受访者担忧被取代，担忧程度随 AI 暴露度上升而增加，早期职业者比资深人士更焦虑。平均生产力评分 5.1/7（显著提升），48% 强调范围扩展，40% 强调速度提升。呈 U 型曲线：被 AI 拖慢和被大幅加速的用户，反而更担忧失业。一位软件工程师说：「以前花几个月做的网站，我 4-5 天就做完了。」

3、Anthropic 工程博客：Scaling Managed Agents — 将大脑与双手解耦 confirmed

Anthropic 详解 Managed Agents 的架构设计。核心理念：将 Session（持久事件日志）、Harness（调用循环）和 Sandbox（执行环境）解耦，类似操作系统的 read() 抽象。此前他们的容器是「宠物」——一旦失败，整个会话丢失。解耦后 p50 首 token 延迟降低约 60%，p95 改善超 90%。Session 不等于 Claude 的上下文窗口，而是通过 getEvents() 切片的外部上下文。MCP + Vault 模式用于安全的自定义工具接入。

4、GPT-5.5 通过 Codex「后门」API 可用，定价公布 confirmed

Simon Willison 介绍了通过 OpenAI Codex 订阅路径使用 GPT-5.5 的方法（公共 API 尚未正式开放），并发布了 llm-openai-via-codex 插件。GPT-5.5 API 定价：输入 5 美元/百万 token，输出 30 美元/百万 token（GPT-5.4 分别为 2.5 和 15 美元）。Pro 版输入 30 美元、输出 180 美元。OpenAI 的 Romain Huet 确认官方支持通过 ChatGPT 订阅在第三方工具中使用 Codex。

5、Claude Design 发布：从设计到原型到交付的 AI 视觉协作工具 confirmed

Anthropic Labs 推出 Claude Design，基于 Claude Opus 4.7，支持设计、原型、演示文稿、单页文档等视觉协作场景。亮点包括设计系统导入（从代码和设计文件中提取）、内联评论、直接编辑、Claude 生成的交互滑块、以及向 Claude Code 的交付包。客户反馈：「我们最复杂的页面在其他工具需要 20+ 次提示，在 Claude Design 只需 2 次。」目前为 Pro/Max/Team/Enterprise 的研究预览版。

📝 文章推荐

1、Fly.io：2026 年，MCP 不是扩展 Agent 的正确方式

Fly.io 为 Sprites 添加了 MCP 支持——但同时发文论证 MCP 在 2026 年是扩展 Agent 的错误默认方案。他们主张 CLI + 可发现 API + Skills（渐进式披露）优于 MCP 的工具描述膨胀。MCP 把所有工具定义塞进上下文，而 Skill 只需一句话描述就能让 Agent 按需调用。文章还痛批「无状态沙箱」，认为 Agent 需要的是持久的、有网络的「真正的计算机」。

2、The Agent Stack Bet：为什么 Agent 的瓶颈不是模型质量

Addy Osmani 认为生产级 Agent 失败的原因通常不是模型，而是管道：会话逻辑、共享凭证、治理缺失。他提出四个押注：(1) 每个 Agent 独立身份而非共享服务账号；(2) 跨系统统一上下文；(3) 持久执行和检查点；(4) 平台化而非自建记忆/可观测/重试栈。核心金句：「A promise in a prompt is not a policy.」

3、Email is Crazy — 邮件基础设施的疯狂架构之旅

一篇从 MSA/MTA/MUA 讲到 SPF/DKIM/DMARC 的邮件系统全景。SMTP 是明文协议，信封和 From 头部不做匹配检查——这是钓鱼攻击的根源。行业的应对不是修复 SMTP，而是不断叠加「胶带」。最恐怖的场景不是退信，而是静默丢弃——RFC 5321 并不禁止这样做。结论：邮件不优雅，但它是 Lindy 的。

4、TigerBeetle：Rust 依赖的工具链兼容性窗口只有约两年

TigerBeetle 的 Rust 客户端因传递依赖（如 syn）在 patch 版本中提升 rust-version 而遭遇 CI 中断。作者对 crates.io 前 100 包做了兼容性二分搜索实验，发现典型 Rust 项目只有约两年的编译器可用窗口。Serde 值得表扬（兼容回溯到 Rust 1.31），futures 因 proc-macro 链拖后腿。TigerBeetle 的 Zig 核心坚持「零依赖」策略。

5、Tailscale CEO：当行业变成龙卷风，我们不制造更多龙卷风

Avery Pennarun 的产品哲学文章：AI 带来的速度是好事，但速度缺乏判断力时，会把粗糙的边角传导给整条链路。他用 VR 蹒跚学步的孩子比喻当前的 AI 狂热——沉浸感很强，但总得有人防止你撞墙。Tailscale 定位自己为「龙卷风中的基础设施」——做连接、身份、策略、日志这些「无聊但致命」的事。

🧩 本期概念库

1. AGI 条款 (AGI Clause)

一句话解释：OpenAI 与微软协议中的特殊条款：一旦 OpenAI 实现 AGI，微软的技术许可将失效。这个条款曾被视为 AI 行业最大的法律悬念，现已随协议修订而实质性消亡。
背景来源：OpenAI-微软协议修订 + Simon Willison 分析
为什么重要：标志着 AI 行业从理想主义阶段进入商业现实主义阶段。「AGI 触发器」曾是约束 AI 公司最独特的法律工具，它的消亡意味着资本和商业逻辑已经完全主导了 AI 的发展方向。
团队应用场景：关注 AI 供应商的商业条款变化——独占协议正在被多云策略取代，我们的 API 接入架构也应具备供应商切换能力。

2. 宠物容器问题 (Pet Container Problem)

一句话解释：Anthropic 在 Managed Agents 中遇到的架构缺陷：Agent 会话绑定到特定容器（像养宠物一样），容器故障则会话丢失。解决方案：将 Session/Harness/Sandbox 解耦为独立抽象层，容器变为可替换的「牛」。
背景来源：Anthropic Engineering: Scaling Managed Agents
为什么重要：所有运行 AI Agent 的团队都会遇到这个问题。有状态的 Agent 需要无状态的执行环境——这与微服务架构的演进路径完全一致。
团队应用场景：审查我们自己的 Agent 运行方式：会话状态是否与执行环境解耦？容器或进程崩溃后，Agent 能否恢复？

3. Agent 按量计费时代

一句话解释：从 GitHub Copilot 的 AI Credits 到 Tailscale Aperture 的跨供应商配额，AI 工具正在集体从固定订阅转向按量计费。根本原因：Agent 式用量（多小时自主运行）与聊天式用量（几秒钟问答）的成本差异可达数个数量级。
背景来源：GitHub Copilot 计费变更 + Tailscale Aperture 公告
为什么重要：这将根本改变团队评估和使用 AI 工具的方式。「免费试用」心态必须转变为「成本意识」心态。
团队应用场景：在 6 月 1 日前建立 AI 工具 token 消耗基线，设置团队和个人级别的用量预警和预算上限。

4. 声纹即不可轮换凭证

一句话解释：与密码不同，声纹一旦泄露就无法「轮换」。Mercor 泄露事件将政府 ID、自拍和高质量录音绑定在一起，为攻击者提供了完整的身份+声纹包。这类数据可用于银行声纹验证绕过、Vishing 和深伪造电话。
背景来源：Mercor 4TB 数据泄露分析
为什么重要：生物特征认证的根本脆弱性：你可以轮换密码和密钥，但不能轮换你的声音和脸。
团队应用场景：验证我们的身份认证和反欺诈系统是否过度依赖单一生物特征。考虑多因素验证中生物特征仅作为辅助而非唯一因子。

5. 渐进式披露 vs MCP 工具膨胀 (Progressive Disclosure vs Tool Bloat)

一句话解释：Fly.io 的观点：MCP 将所有工具定义塞进 Agent 上下文（工具膨胀），而更好的方式是渐进式披露——Agent 只需知道一句话描述就能按需加载完整工具。类比：你不会在每次对话中把整本字典放在桌上。
背景来源：Fly.io: Unfortunately, Sprites Now Speak MCP
为什么重要：随着 Agent 工具数量增长，上下文窗口管理成为关键瓶颈。这个设计哲学将影响所有 Agent 框架的演进方向。
团队应用场景：设计内部 Agent 工具时，优先考虑 Skill/CLI 模式（按需加载），而非一次性注册所有工具。保持 Agent 上下文精简。

当大脑疲劳时，不刷手机、不看视频，而是面壁静坐 5-10 分钟。作者发现这像健身一样——开始时极度抗拒，但坚持后专注力显著恢复。原理：减少微多巴胺刺激，让前额叶皮层重新接管注意力。HN 453 赞。适合下午三点写不动代码的你。

你的蓝色是我的蓝色吗？
